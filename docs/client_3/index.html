Configuration

To use CLI you can neither use flags or create config file which will be used by cli to take common arguments from.

Configuration file

Config file holds privat key of an account and url for a specific network. Profiles are used to connect to the network and use key to sign transactions.

Structure of config file looks like following:

cli:
  private_key: '14F8423888AE5F719B06FF5B5B242DDD31CE8003639FC69A2D3EFB82E32A2FBF'
daemon:
  address: 'http://127.0.0.1'
  port: '8080'

To point cli on config file you can:

  • place it in 'default' place where cli automatically will search for it ( it's ~/.xpx/config_cli.yml )
  • pass custom location using flag config - xpx-cli --config '/root/.xpx/config_cli.yml'

Using configuration flags

If you for some reason want to pass common arguments manually in cli call, you can use following flags:

  • port 8080
  • address 'http://127.0.0.1'
  • key 14F8423888AE5F719B06FF5B5B242DDD31CE8003639FC69A2D3EFB82E32A2FBF

Example:

xpx-cli --port 8080 --address 'http://127.0.0.1' --key 14F8423888AE5F719...9A2D3EFB82E32A2FBF
