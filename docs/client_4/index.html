Commands

Daemon

Commands which can be used for managing daemon process of ProximaX node.

Start

Installs daemon if it's not installed and starts it.

Flags required:

  • daemon-config - absolute path to storage node config

dfms-cli daemon start --daemon-config /somepath/.../config.yml

Stop

Stops daemon if it's running.

dfms-cli daemon stop

Note: stop command does not delete daemon service, if you want permanently delete service you might want to use something like (for Ubuntu):

systemctl stop dfms.service
systemctl disable dfms.service
rm /etc/systemd/system/dfms.service
systemctl daemon-reload
systemctl reset-failed

Status

Get daemon status running/stopped

dfms-cli daemon status

Onboarding

Onboarding operations used to initialize local node in network. These operations should be used only with local node.

Start

Starts onboarding process and returns <PROCESS_TOKEN> which you can use to check onboarding status.

dfms-cli onboarding start

Status

Get the status of onboarding process.

Arguments:

  • <PROCESS_TOKEN> - token which presents some onboarding process.

dfms-cli onboarding status <PROCESS_TOKEN>

Network

Get node's network data.

Type

Get node's network type.

dfms-cli network type

Peers

Get peers public keys within current network.

dfms-cli network peers

Peer

Get detailed information about peers with <PUBLIC_KEYS>.

Arguments:

  • <PUBLIC_KEYS> - public keys of some peer nodes.

dfms-cli network peer <PUBLIC_KEYS>

Contract

Command which can be used to upload\download files to storage or get info about existing contracts.

Create

Upload file to the storage.

Flags allowed:

  • duration - duration of contract in blocks (default 10000)
  • replications - count of replicators, which will store the file (default 3)
  • verifications - count of verifiers, which will verify that replicators store actual file (default 3)
  • verifiers - array of pre-selected verifiers to be used in contract
  • pdp - proof of data posession algorithm (CPOR/MHT/SPDP - default SPDP)

Arguments:

  • <FILE_PATH> - path to file which you want to upload.

xpx-cli contract create <FILE_PATH>
xpx-cli contract create --duration 100 --replications 1 --verifications 1 --pdp MHT test1.png

Download

Download file by contract public key.

Flags allowed:

  • path - path where file should be download Arguments:
  • <CONTRACT_PUBLIC_KEY> - contract public key.

xpx-cli contract download <CONTRACT_PUBLIC_KEY>
xpx-cli contract download 00112233445566778899ff --path filename.pdf

Info

Get info about contract by public key.

Arguments:

  • <CONTRACT_PUBLIC_KEY> - contract public key.

xpx-cli contract info <CONTRACT_PUBLIC_KEY>
